Ltac Inventory Cleanup Status
=============================

Scope
-----

This file tracks the Ltac cleanup pass requested for the current branch.  It is
kept as a live status document: completed cleanup is recorded here, and the only
remaining items are deliberately deferred because they are low-value, interactive
helpers, or require broader proof-style migration that could increase compile
time.

----------------

- Initial scan: 235 `Ltac` / `Tactic Notation` definitions.
- After cleanup: 191 definitions.
- Removed in this pass: 44 unused wrappers, stale normalizers, or old-route
  proof-search helpers.

Completed cleanup
-----------------

(`store_normalize`, `better_store_solver`, `type_open_env_syntax_norm`).

at source with the existing syntax and support normalizers, not keep old

Reason: no call sites.  The connective tactics were a parallel style that the

Underused but important tactics
-------------------------------

These should not be deleted.  They are the right direction for future proof
cleanup:

- `collect_stales`
  - Use this to build forbidden sets for cofinite/open proofs.
  - Avoid hand-written `L := ∅` or manually assembled stale sets.
- `auto_exists_L`
  - Useful when a proof goal asks for a cofinite set.
- `econstructor_L` / `econstructor_L_specialized`
  - Encodes the constructor-with-fresh-name pattern.
  - Should be considered in BasicTyping/Open and Soundness proofs before
    writing custom fresh-name boilerplate.
- `soundness_case_start`
  - Good shared prelude for fundamental cases.
  - Expand only with deterministic, cheap steps.
- `denotation_regular`
  - Useful idea: extract repo-specific resource facts before calling stdpp.
  - Extend by adding regular lemmas, not by broad proof search.
