Soundness case bridge implementation plan
=========================================

Principle
---------

The Fundamental proof should look like the proven TLet branch:

1. The Soundness case consumes context denotation and induction hypotheses.
2. It derives concrete `denot_ty_lvar_gas` premises for the current world.
3. It calls a Denotation-level helper such as `lam_intro_denotation`.

Denotation-level helpers should not mention `denot_ctx_under`, `Γ`, or typing
induction hypotheses.  They may still mention basic typing and operational
premises when those are genuinely part of the semantic case.

Case-by-case plan
-----------------

1. AppOp

- Already has a direct helper, `appop_intro_denotation`, whose proof is complete.
- Keep the Soundness bridge proved by extracting the argument IH and calling
  `appop_intro_denotation`.

2. Lam

- Soundness bridge:
  - pick/receive the fresh parameter from the Arrow forall proof obligation;
  - build the context denotation for `CtxComma Γ (CtxBind y τx)`;
  - call the body IH to get
    `denot_ty_lvar_gas ... (<[LVFree y := erase_ty τx]> Σ) ({0 ~> y} τ) (e ^^ y)`;
  - call `lam_intro_denotation`.
- Needed bridge lemma:
  `denot_ctx_under_comma_bind_from_arg_denotation`.

3. LamD

- Same as Lam, but the body context is `CtxStar Γ (CtxBind y τx)`.
- Need a separating bridge:
  `denot_ctx_under_star_bind_from_arg_denotation`.
- This will use existing `FStar`/resource product lemmas.

4. App

- Extract function and argument IHs on the same world.
- Use the Arrow denotation's forall witness and transport the fresh witness
  back to the actual argument variable.
- Needed helper family:
  - argument fvar-to-witness transport;
  - result open/rename transport.

5. AppD

- Split the `CtxStar Γ1 Γ2` context into function and argument resources.
- Use function IH on the function resource and argument IH on the argument
  resource.
- Eliminate the Wand denotation and transport the fresh witness as in App.

6. LetD

- Split `CtxStar Γ1 Γ2`.
- Use IH1 to build the result extension for `e1`.
- Combine the result-extension world with the Γ2 resource to satisfy the body
  context `CtxStar Γ2 (CtxBind x τ1)`.
- Call `letd_intro_denotation`.

7. Fix

- Denotation helper should follow HATs' `denotation_application_fixed` shape.
- The body premise is not merely a recursive IH; it says that after opening the
  ordinary argument `y`, the opened body denotes a function expecting recursive
  self:

    `CTArrow (fix_rec_call_ty b y τx τ) ({0 ~> y} τ)`

- The Denotation proof will:
  - use `constant_lt_for_base_well_founded b`;
  - perform well-founded induction on the current base constant;
  - build the recursive self denotation from the induction hypothesis.

8. FixD

- Same HATs-shaped recursive helper as Fix.
- The Soundness bridge builds the body context with `CtxStar`.

9. MatchBoth

- Split the `CtxSum` context via `FPlus`.
- Call the two scrutinee IHs and the two branch IHs on their corresponding
  branch worlds.
- Call `match_both_intro_denotation`.

10. MatchTrue / MatchFalse

- The Denotation-level helper must receive an unreachable premise.
- The Soundness bridge turns typing-level `branch_unreachable` into a current
  world formula:

    `m ⊨ FImpl (denot_ty_lvar_gas (cty_depth (bool_precise_ty b)) Σ
          (bool_precise_ty b) (tret v)) FFalse`

- Then it calls `match_true_intro_denotation` or
  `match_false_intro_denotation`.

Implementation order
--------------------

0. Redesign context denotation so every recursive call first fixes the ambient
   erased context as a fiber:

   `denot_ctx_under Σ Γ :=
      FFibVars (lvars_of_atoms (dom Σ))
        (FAnd (basic_world_formula (atom_env_to_lty_env (erase_ctx_under Σ Γ)))
          (match Γ with ... end))`.

   This keeps the denotation entirely inside Context Logic while making the
   intended invariant explicit: every `dom Σ` fiber of a world satisfying
   `denot_ctx_under Σ Γ` must satisfy the context body after substituting that
   fiber projection.  It does *not* imply that the ambient world itself has a
   singleton projection on `dom Σ`.  The recursive call for
   `CtxComma Γ1 Γ2` is simply

     `denot_ctx_under (Σ ∪ erase_ctx Γ1) Γ2`

   because that call itself adds the fiber for the enlarged ambient context.
   No extra local `FFibVars (dom Γ1)` wrapper is needed.

   The plain context denotation should become:

     `denot_ctx Γ := denot_ctx_under ∅ Γ`.

0a. Add/restore the Logic/Algebra support needed by the new outer fiber.

   Expected helper families:

   - Fiber projection singleton:
     `res_fiber_from_projection ... -> res_restrict mfib X = singleton_world σ`.
     Current concrete wrappers:
       `res_restrict_fiber_from_projection_dom_singleton`,
       `res_restrict_fiber_from_projection_eq_singleton`.
     The empty special case is also available:
       `res_fiber_from_projection_empty_self`,
       `res_fiber_from_projection_empty_eq`.
   - Fiber nesting/idempotence:
     consecutive fibers should be represented as one fiber over the union of
     the two domains.  Do not avoid overlap by projecting the second step over
     `Y ∖ X`; prove the overlap-aware version instead.
     Current Logic-facing obligation:
       `res_models_FFibVars_nested_union_iff`:
         `m ⊨ FFibVars D1 (FFibVars D2 φ) <->
          m ⊨ FFibVars (D1 ∪ D2) φ`.
     This is now the primary theorem.  Prove it by unfolding the `FFibVars`
     model rule, composing the two projection stores, and transporting
     `formula_msubst_store` through the composed projection.  If a direct proof
     needs an algebra fact, keep that fact about consecutive fiber projections
     in Algebra and expose only this Logic theorem to Denotation/Typing.
     The implementation is split into the two reviewable directions
     `res_models_FFibVars_nested_union_l` and
     `res_models_FFibVars_nested_union_r`; the iff lemma is just the pair of
     those two directions.
     The algebra projection support currently has the same shape, but with
     overlap handled by compatible/restricted projection stores:
       `res_fiber_from_projection_nested_union_l`
       `res_fiber_from_projection_nested_union_r`
     Expected forward shape:
       `res_fiber_from_projection m X σX mfibX ->
        res_fiber_from_projection mfibX Y σY mfibXY ->
        storeA_compat σX σY ->
        res_fiber_from_projection m (X ∪ Y) (σX ∪ σY) mfibXY`.
     The compatibility premise is derivable for actual consecutive
     projections by the proved helper:
       `res_fiber_from_projection_store_compat`
     (abstract version `resA_fiber_from_projection_store_compat`).
     Current state: both overlap-aware directions
       `resA_fiber_from_projection_nested_union_l`
       `resA_fiber_from_projection_nested_union_r`
     are proved at the abstract Algebra level; the concrete `res_*` versions
     are thin wrappers.
     Expected reverse shape:
       `res_fiber_from_projection m (X ∪ Y) σXY mfibXY ->
        exists σX mfibX σY,
          σX = storeA_restrict σXY X /\
          σY = storeA_restrict σXY Y /\
          res_fiber_from_projection m X σX mfibX /\
          res_fiber_from_projection mfibX Y σY mfibXY`.
     In the Logic proof, the second projection domain will often be the
     residual domain generated by `formula_msubst_store_fibvars`, e.g.
     `lvars_fv (D2 ∖ lvars_of_atoms (dom σX))`, because the outer
     substitution has already fixed the overlapping part.  This is still an
     instance of the general overlap-aware algebra lemma; the algebra statement
     itself should not bake in a difference.
     These are deliberately about projection composition only; formula-level
     substitution composition stays in `FormulaSyntax`.
     The small proved support lemma
       `res_fiber_from_projection_world_dom`
     records that taking a fiber does not change the ambient world domain; use
     it for scoped/fv obligations after moving from `m` to `mfib`.
     Current syntax-side composition lemma:
       `formula_msubst_store_union`:
         compatible store domains imply
         `formula_msubst_store σ2 (formula_msubst_store σ1 φ) =
          formula_msubst_store (σ1 ∪ σ2) φ`.
     This formula lemma is proved by structural induction, with the only
     remaining primitive obligation being qualifier-level composition:
       `lqual_mlsubst_union`.
     Strengthen this qualifier lemma with an overlap-compatible premise:
       `storeA_compat ρ1 ρ2 ->
        lqual_mlsubst ρ2 (lqual_mlsubst ρ1 q) =
        lqual_mlsubst (ρ1 ∪ ρ2) q`.
     Try exact equality first via `logic_qualifier_ext`; only fall back to a
     models-level/denote transport if dependent `LWorldOn` paths become
     genuinely unmanageable.
     Current state: the exact equality wrapper is proved.  The remaining named
     primitive is the expected path equation for the two back-construction
     routes:
       `lworld_on_mlsubst_back_union`.
     This is a private proof helper for `lqual_mlsubst_union`, not a public
     interface.  Its domain mentions `(D ∖ dom ρ1)` only because the type of
     `lworld_on_mlsubst_back` forces that intermediate domain when composing
     two back substitutions.  Do not generalize it unless another proof needs
     the same construction; public clients should use only `lqual_mlsubst_union`.
     The old subset-shaped `res_models_FFibVars_nested_elim` must remain only a
     thin corollary using union plus same-fv transport for `FFibVars`.
     Current same-fv transport:
       `res_models_FFibVars_same_fv`:
         `lc_lvars D2 ->
          lvars_fv D1 = lvars_fv D2 ->
          m ⊨ FFibVars D1 φ -> m ⊨ FFibVars D2 φ`.
     The explicit `lc_lvars D2` premise is necessary because equal free-variable
     sets do not rule out extra bound lvars in `D2`.
     The same helper now gives the proved wrapper:
       `res_models_FFibVars_idemp_elim`.
   - Singleton fiber connective splitting:
     the old unconditional distributivity obligations
       `res_models_FFibVars_star_distrib_l`
       `res_models_FFibVars_plus_distrib_l`
     are too strong after `FFibVars` started substituting the chosen projection
     store into the body.  `FFibVars D (FStar φ ψ)` only says that every
     `D`-projection fiber can be split; if the ambient world has more than one
     possible `D` projection, the splits for different fibers need not assemble
     into one global split of `m`.

     Replace those obligations by singleton-projection versions.  First add
     Algebra/Logic helpers saying that a singleton projection makes the
     `FFibVars` witness unique:

       `res_fiber_singleton_projection_inv`:
         `dom σD = X ->
          res_restrict m X = singleton_world σD ->
          res_fiber_from_projection m X σ mfib ->
          σ = σD /\ mfib = m`.

       `res_models_FFibVars_singleton_elim`:
         `dom σD = lvars_fv D ->
          res_restrict m (lvars_fv D) = singleton_world σD ->
          m ⊨ FFibVars D φ ->
          m ⊨ formula_msubst_store σD φ`.

       `res_models_FFibVars_singleton_intro`:
         `lc_lvars D ->
          dom σD = lvars_fv D ->
          res_restrict m (lvars_fv D) = singleton_world σD ->
          m ⊨ formula_msubst_store σD φ ->
          m ⊨ FFibVars D φ`.

     The first useful algebra helpers are product/sum inheritance helpers.
     These are proved and remain useful when the branch worlds already contain
     the fixed domain:

       `res_product_le_singleton_restrict_inv`:
         `res_product m1 m2 Hc ⊑ m ->
          X ⊆ world_dom (m1 : WorldT) ->
          X ⊆ world_dom (m2 : WorldT) ->
          res_restrict m X = singleton_world σ ->
          res_restrict m1 X = singleton_world σ /\
          res_restrict m2 X = singleton_world σ`.

       `res_sum_le_singleton_restrict_inv`:
         same shape for `res_sum`.

     However, this is not enough for the shared eliminator after `mlsubst`.
     The reason is subtle but important: substituting the singleton store into
     `FFibVars D φ` rewrites it to a residual fiber over
     `D ∖ lvars_of_atoms (dom σD)`, whose free-variable projection is empty
     when `dom σD = lvars_fv D`.  Therefore the star/plus split of
     `formula_msubst_store σD (FFibVars D φ)` can return components whose
     worlds do not contain `lvars_fv D` at all.  We cannot simply invert
     singleton projection on those components.

     The next required algebra fact is a singleton pullback/lift operation:
     if `m` has a singleton projection `σD` on `X`, and a component `mi`
     produced by a product/sum split is below `m`, construct an enlarged
     component `miD` that adds exactly this singleton `X` projection while
     remaining below `m`.  The two enlarged components should still be
     compatible/summable because they share the same singleton overlap.

      `res_product_le_singleton_pullback` (planned):
         from `res_product m1 m2 Hc ⊑ m` and
         `res_restrict m X = singleton_world σX`, construct `m1D`, `m2D`,
         `HcD` such that
           `res_product m1D m2D HcD ⊑ m`,
           `res_restrict m1D X = singleton_world σX`,
           `res_restrict m2D X = singleton_world σX`,
           and `m1D/m2D` agree with `m1/m2` on the old branch formula support.

        Use the concrete construction

          `m1D := res_restrict m (world_dom (m1 : WorldT) ∪ X)`
          `m2D := res_restrict m (world_dom (m2 : WorldT) ∪ X)`.

        This construction is chosen because both enlarged worlds are directly
        obtained from the ambient `m`, so the final `⊑ m` proof is a restrict
        proof rather than a reconstruction of arbitrary ambient stores.  It
        should provide:

          `res_restrict m1D (world_dom (m1 : WorldT)) = m1`
          `res_restrict m2D (world_dom (m2 : WorldT)) = m2`

        using `m1 ⊑ res_product m1 m2 Hc ⊑ m` and the corresponding right
        component fact, and:

          `res_restrict m1D X = singleton_world σX`
          `res_restrict m2D X = singleton_world σX`

        by commuting nested restrictions with
        `res_restrict m X = singleton_world σX`.

        Compatibility of `m1D` and `m2D` is the only real algebraic point.
        Their overlap is
        `(world_dom m1 ∪ X) ∩ (world_dom m2 ∪ X)`.  Agreement on the old
        `world_dom m1 ∩ world_dom m2` part follows from the original
        `world_compat m1 m2` after restricting `m` back to `m1`/`m2`; agreement
        on the `X` part follows from the singleton projection.  Package this
        either as part of the pullback proof or as a smaller helper:

          `world_compat_restrict_union_singleton_overlap`.

        After this lemma, branch model transport is:

          `n1 ⊨ θ`
          `formula_fv θ ⊆ world_dom (n1 : WorldT)`
          `res_restrict m1D (world_dom (n1 : WorldT)) = n1`
          ------------------------------------------------
          `m1D ⊨ θ`

        using `res_models_minimal_on` / `res_models_kripke`.

      `res_sum_le_singleton_pullback` (planned):
         analogous for `res_sum`.

        For sum, prefer reusing the existing
        `res_sum_lift_along_restrict`/slice-pullback infrastructure when
        possible.  If the lifted worlds become full-domain subworlds of `m`,
        derive the singleton projection by a small subset lemma:

          `res_subset_singleton_restrict`:
            `res_subset p m ->
             X ⊆ world_dom (p : WorldT) ->
             res_restrict m X = singleton_world σX ->
             res_restrict p X = singleton_world σX`.

        The product route cannot use full-domain pullback directly because two
        full-domain subworlds need not be compatible unless the whole ambient
        world is singleton; hence the narrower `world_dom ni ∪ X` construction.

     The Logic-facing replacement for the old shared eliminator is then:

       `res_models_FFibVars_star_elim_shared_singleton`:
         `dom σD = lvars_fv D ->
          res_restrict m (lvars_fv D) = singleton_world σD ->
          m ⊨ FFibVars D (FStar (FFibVars D φ) (FFibVars D ψ)) ->
          exists m1 m2 Hc,
            res_product m1 m2 Hc ⊑ m /\
            m1 ⊨ FFibVars D φ /\
            m2 ⊨ FFibVars D ψ`.

       `res_models_FFibVars_plus_elim_shared_singleton`:
         analogous for `FPlus`.

     Revised proof shape for star:
       1. use `res_models_FFibVars_singleton_elim` to turn the outer
          `FFibVars` into
          `m ⊨ formula_msubst_store σD (FStar ...)`;
       2. simplify the substitution through `FStar`;
       3. use `res_models_star_iff` to obtain `m1`, `m2`;
       4. use the product singleton pullback lemma to enlarge these components
          to `m1D`, `m2D` carrying the fixed projection;
       5. use the branch support/agreement part of the pullback lemma to
          transport the already obtained branch models from `m1/m2` to
          `m1D/m2D`;
       6. rewrite `formula_msubst_store_fibvars`, use empty/residual-fv
          normalization and `res_models_FFibVars_singleton_intro` to rebuild
          `m1D ⊨ FFibVars D φ` and `m2D ⊨ FFibVars D ψ`.
     The plus proof is the same with `res_models_plus_iff` and sum inheritance.

     Denotation should not call an unconditional
     `denot_ctx_under_star_elim`.  Replace it by a singleton-aware eliminator:

       `denot_ctx_under_star_elim_singleton`:
         `dom σΣ = dom Σ ->
          res_restrict m (dom Σ) = singleton_world σΣ ->
          m ⊨ denot_ctx_under Σ (CtxStar Γ1 Γ2) ->
          exists m1 m2 Hc,
            res_product m1 m2 Hc ⊑ m /\
            m1 ⊨ denot_ctx_under Σ Γ1 /\
            m2 ⊨ denot_ctx_under Σ Γ2`.

     The analogous `denot_ctx_under_sum_elim_singleton` handles `CtxSum`.
     Upper proofs obtain the singleton premise only after choosing a concrete
     outer `FFibVars (ctx_base_vars Σ)` fiber of `denot_ctx_under`; it is not a
     consequence of an arbitrary ambient `m ⊨ denot_ctx_under Σ Γ`.

     Current proved Denotation helpers:

       `denot_ctx_under_fiber_from_projection`:
         `res_fiber_from_projection m (dom Σ) σΣ mfib ->
          m ⊨ denot_ctx_under Σ Γ ->
          mfib ⊨ denot_ctx_under Σ Γ`.

       `denot_ctx_under_star_elim_in_fiber` / `...sum...`:
         split a chosen `dom Σ` fiber `mfib`, not the original ambient world.

     Do not try to prove either of the following false routes:

       `res_fiber_from_projection m X σ mfib -> mfib ⊑ m`
       `res_restrict m X = singleton_world σ -> m ⊨ φ ->
        m ⊨ formula_msubst_store σ φ`.

     The first is false because `⊑` is raw restriction equality; since a fiber
     has the same domain as its source, it would imply `mfib = m`.  The second
     is false because substituting the projection store changes the formula's
     interpretation.  Star/sum/appd/letd soundness must therefore be proved by
     a fiberwise bridge: choose each base fiber from `denot_ctx_under`, prove
     the corresponding denotation inside that fiber, and rebuild the required
     top-level denotation using a Denotation-specific fiberwise introduction
     lemma, not by transporting arbitrary formulas through `mlsubst`.
   - Models-level `FFibVars` connective helpers, especially for `FAnd`,
     `FStar`, and `FPlus`.  Avoid formula equality when `formula_msubst_store`
     would make equality too strong; prefer `m ⊨ ... -> ...` / iff lemmas.
     Completed helpers:
       `formula_msubst_store_empty`,
       `res_models_FFibVars_empty_iff`,
       `res_models_FFibVars_and_l`,
       `res_models_FFibVars_and_r`.
     The old unconditional star/plus distributivity lemmas have been deleted.
     The only supported route is through the singleton-projection eliminators
     above; upper proofs must expose the outer context fiber before splitting.

   The context basic-world projection has been moved to the BasicDenotation
   layer as:

     `basic_world_formula_fibvars_elim`.

   Prove easy directions immediately.  Keep hard algebra facts as named
   admitted obligations only if needed to wire the new denotation through the
   context bridges.

   During migration, delete the old compatibility alias `denot_ctx_in_env`
   entirely if it still exists.  The live context denotation should be just
   `denot_ctx_under`.

1. Keep the current `ctx_sub_under` meaning: it restricts the current world to
   the judgment support `X` and then satisfies the target context.  Prove the
   Fundamental case through one Denotation-level transport lemma:

   `denot_ty_in_ctx_under_ctx_sub_transport`.

   This helper packages the two facts that otherwise make the case noisy:
   denotation only observes `fv_tm e ∪ fv_cty τ`, and erased environments that
   agree on that support give the same type denotation.

2. Move context bridge lemmas out of `ContextTyping/theories/Soundness.v` and
   into the Denotation layer.  Soundness should only instantiate IHs and call
   these bridges.  The bridge family is:

   - `denot_ctx_under_comma_bind_from_arg_denotation`
   - `denot_ctx_under_star_bind_from_arg_denotation`
   - `denot_ctx_under_star_elim_singleton`
   - `denot_ctx_under_sum_elim_singleton`
   - `denot_ty_in_ctx_under_comma_bind_to_lvar_insert`
   - `denot_ty_in_ctx_under_star_bind_to_lvar_insert`
   - `denot_ty_lvar_gas_star_union_to_ctx`
   - `basic_typing_star_union_lty_env`
   - `letd_body_world_compat`

   The result-extension bridges should also live below Soundness, but their
   statements must avoid importing the typing judgment into Denotation.  They
   should be phrased in terms of semantic/basic premises, and Soundness should
   derive those premises from `context_typing_wf`.

2a. Strengthen the well-formedness side condition used by typing.

   The old `context_typing_wf` was only a packed regularity definition:

     `wf_context_ty_under Σ Γ τ /\ erase_ctx_under Σ Γ ⊢ₑ e ⋮ erase_ty τ`.

   Keep that shape as `context_typing_regular`, but replace
   `context_typing_wf` by an inductive predicate whose type side condition is
   syntax-directed.  In particular, the `CTWand τx τr` case must record
   `basic_context_ty (dom Σ) τx`, not merely
   `basic_context_ty (dom Σ ∪ ctx_dom Γ) τx`.  This is the invariant needed by
   `CtxStar Γ (CtxBind y τx)`: a disjoint function parameter may not depend on
   the captured context.

   Prove projection lemmas immediately:

   - `context_typing_wf_regular`
   - `context_typing_wf_basic_typing`
   - `context_typing_wf_basic_context_ty_erased`
   - `context_typing_wf_wand_arg_global`

   Existing consumers should use these projections instead of destructing
   `context_typing_wf` as the old pair.

3. Prove easy erased-environment transport bridges immediately.  These are
   mostly `erase_ctx_under` equality/agreement plus
   `atom_env_to_lty_env_insert_free`.

4. Prove `star_elim` and `sum_elim` by unfolding `denot_ctx_under`, applying
   `res_models_star_iff` / `res_models_plus_iff`, and pushing the outer
   `basic_world_formula` assumptions down with `basic_world_formula_subenv`.

5. Prove the argument-bind context bridges using a general preservation lemma:
   if a world extension or product does not remove the relevant context support,
   old context formulas remain true, and the new bind comes directly from the
   supplied argument denotation.

6. Leave genuinely semantic case lemmas in
   `Denotation/theories/ContextTypeDenotationCases.v` as named obligations until
   their individual case proof is tackled.  Do not replace them with old
   wrapper stacks.

7. Strengthen the direct Denotation case helpers with an explicit guard premise
   where needed.

   The robust split is:

   - Soundness derives the *static* denotation guard

       `context_ty_wf ∧ basic_world ∧ expr_basic_typing`

     from `context_typing_wf` and `denot_ctx_under`.
   - Each Denotation-level helper proves the genuinely semantic/operational
     part, especially `expr_total_formula`, for its own case.

   This avoids making App/Match helpers assume their own totality result while
   still removing hidden dependencies on the old packed `context_typing_wf`.
   For value-introduction helpers such as `lam_intro_denotation`,
   `lamd_intro_denotation`, `fix_intro_denotation`, and
   `fixd_intro_denotation`, a later cleanup may strengthen the static guard to a
   zero-gas guard once the generic `tret` totality lemma is in place.
