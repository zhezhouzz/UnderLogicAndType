Soundness case bridge implementation plan
=========================================

Principle
---------

The Fundamental proof should look like the proven TLet branch:

1. The Soundness case consumes `denot_ctx_in_env` and induction hypotheses.
2. It derives concrete `denot_ty_lvar_gas` premises for the current world.
3. It calls a Denotation-level helper such as `lam_intro_denotation`.

Denotation-level helpers should not mention `denot_ctx_in_env`, `Γ`, or typing
induction hypotheses.  They may still mention basic typing and operational
premises when those are genuinely part of the semantic case.

Case-by-case plan
-----------------

1. AppOp

- Already has a direct helper, `appop_intro_denotation`, whose proof is complete.
- Keep the Soundness bridge proved by extracting the argument IH and calling
  `appop_intro_denotation`.

2. Lam

- Soundness bridge:
  - pick/receive the fresh parameter from the Arrow forall proof obligation;
  - build the context denotation for `CtxComma Γ (CtxBind y τx)`;
  - call the body IH to get
    `denot_ty_lvar_gas ... (<[LVFree y := erase_ty τx]> Σ) ({0 ~> y} τ) (e ^^ y)`;
  - call `lam_intro_denotation`.
- Needed bridge lemma:
  `denot_ctx_in_env_comma_bind_from_arg_denotation`.

3. LamD

- Same as Lam, but the body context is `CtxStar Γ (CtxBind y τx)`.
- Need a separating bridge:
  `denot_ctx_in_env_star_bind_from_arg_denotation`.
- This will use existing `FStar`/resource product lemmas.

4. App

- Extract function and argument IHs on the same world.
- Use the Arrow denotation's forall witness and transport the fresh witness
  back to the actual argument variable.
- Needed helper family:
  - argument fvar-to-witness transport;
  - result open/rename transport.

5. AppD

- Split the `CtxStar Γ1 Γ2` context into function and argument resources.
- Use function IH on the function resource and argument IH on the argument
  resource.
- Eliminate the Wand denotation and transport the fresh witness as in App.

6. LetD

- Split `CtxStar Γ1 Γ2`.
- Use IH1 to build the result extension for `e1`.
- Combine the result-extension world with the Γ2 resource to satisfy the body
  context `CtxStar Γ2 (CtxBind x τ1)`.
- Call `letd_intro_denotation`.

7. Fix

- Denotation helper should follow HATs' `denotation_application_fixed` shape.
- The body premise is not merely a recursive IH; it says that after opening the
  ordinary argument `y`, the opened body denotes a function expecting recursive
  self:

    `CTArrow (fix_rec_call_ty b y τx τ) ({0 ~> y} τ)`

- The Denotation proof will:
  - use `constant_lt_for_base_well_founded b`;
  - perform well-founded induction on the current base constant;
  - build the recursive self denotation from the induction hypothesis.

8. FixD

- Same HATs-shaped recursive helper as Fix.
- The Soundness bridge builds the body context with `CtxStar`.

9. MatchBoth

- Split the `CtxSum` context via `FPlus`.
- Call the two scrutinee IHs and the two branch IHs on their corresponding
  branch worlds.
- Call `match_both_intro_denotation`.

10. MatchTrue / MatchFalse

- The Denotation-level helper must receive an unreachable premise.
- The Soundness bridge turns typing-level `branch_unreachable` into a current
  world formula:

    `m ⊨ FImpl (denot_ty_lvar_gas (cty_depth (bool_precise_ty b)) Σ
          (bool_precise_ty b) (tret v)) FFalse`

- Then it calls `match_true_intro_denotation` or
  `match_false_intro_denotation`.

Implementation order
--------------------

1. Keep the current `ctx_sub_under` meaning: it restricts the current world to
   the judgment support `X` and then satisfies the target context.  Prove the
   Fundamental case through one Denotation-level transport lemma:

   `denot_ty_in_ctx_under_ctx_sub_transport`.

   This helper packages the two facts that otherwise make the case noisy:
   denotation only observes `fv_tm e ∪ fv_cty τ`, and erased environments that
   agree on that support give the same type denotation.

2. Move context bridge lemmas out of `ContextTyping/theories/Soundness.v` and
   into the Denotation layer.  Soundness should only instantiate IHs and call
   these bridges.  The bridge family is:

   - `denot_ctx_in_env_comma_bind_from_arg_denotation`
   - `denot_ctx_in_env_star_bind_from_arg_denotation`
   - `denot_ctx_in_env_star_elim`
   - `denot_ctx_in_env_sum_elim`
   - `denot_ty_in_ctx_under_comma_bind_to_lvar_insert`
   - `denot_ty_in_ctx_under_star_bind_to_lvar_insert`
   - `denot_ty_lvar_gas_star_union_to_ctx`
   - `basic_typing_star_union_lty_env`
   - `letd_body_world_compat`

   The result-extension bridges should also live below Soundness, but their
   statements must avoid importing the typing judgment into Denotation.  They
   should be phrased in terms of semantic/basic premises, and Soundness should
   derive those premises from `context_typing_wf`.

3. Prove easy erased-environment transport bridges immediately.  These are
   mostly `erase_ctx_under` equality/agreement plus
   `atom_env_to_lty_env_insert_free`.

4. Prove `star_elim` and `sum_elim` by unfolding `denot_ctx_in_env`, applying
   `res_models_star_iff` / `res_models_plus_iff`, and pushing the outer
   `basic_world_formula` assumptions down with `basic_world_formula_subenv`.

5. Prove the argument-bind context bridges using a general preservation lemma:
   if a world extension or product does not remove the relevant context support,
   old context formulas remain true, and the new bind comes directly from the
   supplied argument denotation.

6. Leave genuinely semantic case lemmas in
   `Denotation/theories/ContextTypeDenotationCases.v` as named obligations until
   their individual case proof is tackled.  Do not replace them with old
   wrapper stacks.
