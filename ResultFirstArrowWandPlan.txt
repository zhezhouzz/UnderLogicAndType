# Result-First Arrow/Wand Denotation Plan

## Goal

Change `CTArrow` and `CTWand` denotation so function-typed terms are handled
like `CTSum`: first bind each produced function value with a result graph, then
prove the function-value specification for `ret f`.

This is a semantic change, not a proof-script workaround.  It prevents the TLet
Arrow/Wand proof from trying to commute the let-result extension through a
fresh argument world, which is false when the result of `e1` and the argument
resource are correlated.

## Definition Shape

Introduce value-level helpers in `Denotation/theories/TypeDenote.v`:

```coq
arrow_value_denote_gas gas Σ τx τr ef
wand_value_denote_gas gas Σ τx τr ef
```

`ty_denote_gas` uses an outer `FForall` result binder for Arrow/Wand:

```coq
FForall
  (FImpl
    (expr_result_formula_at (lvars_shift_from 0 (dom Σg))
      (tm_shift 0 e) (LVBound 0))
    (arrow_value_denote_gas gas' Σf
      (cty_shift 0 τx) (cty_shift 1 τr) (tret (vbvar 0))))
```

and analogously for `wand_value_denote_gas`.  After opening both binders, the
function value is `vbvar 1`, the argument is `vbvar 0`, and the application is:

```coq
tapp_tm (tret (vbvar 1)) (vbvar 0)
```

## Proof Recovery Order

1. Repair `TypeDenote.v`: open-env, singleton-open, env-agree, lvars/fv/scope.
2. Repair value-intro/value-elim helpers for Lam/LamD/App/AppD/Fix.
3. Repair `TypeEquivTLet.v`: Arrow/Wand use outer result correspondence first,
   then recurse only on the value-level result body.
4. Restore `tlet_intro_denotation` as an iff.
