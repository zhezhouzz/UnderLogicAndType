Freshness Regular Analysis
==========================

Goal
----

Track freshness-related `assert` / `pose proof` patterns in Denotation and
Soundness proofs, then replace repeated local stdpp reasoning with regular
lemmas extracted from repo definitions. Pure stdpp hypotheses such as
`x ∉ A ∪ B` are left to the folder-local solver; regular lemmas are only for
facts that pass through our definitions.

Scan Commands
-------------

```
rg -n "assert \([^\\n]*(∉|fresh|dom|##)|pose proof .*fresh|pose proof .*notin|pose proof .*dom" \
  Denotation/theories ContextTyping/theories/Soundness*.v
```

Main Buckets
------------

1. Fresh-for destructuring
   Many proofs choose `fresh_for (...)` and immediately assert membership
   exclusions for each component. Most of this is pure set reasoning and should
   be handled by local `*_fresh_solve`; only repo-definition projections should
   become regular lemmas.

2. Inserted lty-env freshness
   Repeated shape:
   `LVFree z ∉ dom (<[LVFree y := T]> (atom_env_to_lty_env (erase_ctx Γ)))`.
   Current proofs manually rewrite `dom_insert_L`, `atom_store_to_lvar_store_dom`,
   and then use `ctx_erasure_under_erase_ctx_dom_subset`.

   General regular lemma planned/implemented:
   `ctx_erasure_under_inserted_erase_ctx_lty_env_fresh`.

3. Result-alias freshness after opening a result type
   Repeated shape:
   `z ∉ fv_value vf ∪ {[y]} ∪ fv_cty (cty_open 0 y τ)`.
   This is always derived from a stronger freshness hypothesis plus
   `cty_open_fv_subset`. This should be a syntax-level helper:
   `value_open_result_alias_fresh`.

4. Relevant-env value freshness
   Repeated use of
   `soundness_relevant_env_arrow_value_fresh` and
   `soundness_relevant_env_wand_value_fresh`.
   These are already good regular lemmas. Next optimization is to make
   `soundness_fresh_solve` solve their side conditions at call sites.

5. Resource extension/domain freshness
   Repeated uses of `res_extend_by_dom`, `res_extend_by_output_fresh`,
   and world-domain side conditions. `DenotationSetMapFacts.v` already has
   `denotation_regular`; remaining local asserts should be migrated gradually.
