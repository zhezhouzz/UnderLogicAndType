TypeEquivBody / FiberBaseCore Refactor Analysis
===============================================

Scope
-----

Files inspected:

- `Denotation/theories/TypeEquivBody.v`
- `Denotation/theories/TypeEquivFiberBaseCore.v`
- `Denotation/theories/TypeEquivFiberBody.v`

Findings
--------

1. `TypeEquivBody.v` Over/Under/Sum wrappers repeat the same conversion:

   ```
   typed_total_equiv_on ... -> typed_fiber_equiv_on ...
   ```

   This conversion already exists as `typed_fiber_equiv_of_tm_equiv` in
   `TypeEquivTermBase.v`, but the wrappers were still manually calling
   `typed_fiber_equiv_intro`, `tm_equiv_on_to_fiber_equiv`, and the source/target
   zero accessors.

2. `typed_total_equiv_inter_l/r` and `typed_total_equiv_union_l/r` repeat the
   same record projection pattern: keep term equivalence and total equivalence,
   then project the two gas-zero facts with `ty_denote_gas_zero_*`.

   This should be a generic `typed_total_equiv_project_context` lemma over
   `context_ty_lvars`, erasure equality, and `context_ty_shape_ok`.

3. `TypeEquivFiberBaseCore.v` left/right lemmas for Inter/Union/Sum also share
   the same proof skeleton around `ty_denote_gas_zero_project_context`.

   This is more delicate because the right projection needs shape/erasure facts
   extracted from the larger type's guard. Leave the public lemma names as-is
   for now; future optimization can add small shape extractors.

4. `TypeEquivFiberBody.v` Over/Under body proofs are genuinely duplicated, but
   they differ in the polarity connective (`FOver` vs `FUnder`) and in the
   context type used by the projected result relation. A useful future
   abstraction would parameterize over:

   - the context type (`CTOver b Ď†` / `CTUnder b Ď†`),
   - the body formula constructor (`FOver` / `FUnder`),
   - the FV observation lemma (`formula_fv_open_*_body_obs`).

   This should be done carefully because the current proofs are long but stable.

Implementation Status
---------------------

Batch A:

- Replace manual typed-total-to-fiber conversions in `TypeEquivBody.v` with
  `typed_fiber_equiv_of_tm_equiv`.
- Add `typed_total_equiv_project_context` and rewrite Inter/Union typed-total
  projection lemmas through it.
- Add `ty_denote_gas_zero_shape_ok` in `TypeEquivFiberBaseCore.v` and use it
  in Inter/Union/Sum left/right zero projections.

Deferred:

- Generalize `TypeEquivFiberBody.v` Over/Under body proof into one parametric
  lemma. This is higher risk and should be done with a local proof scratch first.
- Factor `TypeEquivFiberBaseCore.v` left/right zero projection proofs with
  shape extractors.
