Lemma dependency graph for Denotation and ContextTyping
=========================================================

Status
------

This file records the current proof route after the Context Logic / Context
Type refactor.  The graph is based on source-level dependencies in the proof
scripts, not on a kernel-level proof-term extraction.  It should be maintained
as a design map: when a high-level lemma starts depending on a wrapper whose
only purpose was legacy compatibility, that wrapper should either be removed or
marked here as intentionally kept.

Current unchecked proof nodes:

- None in the core ContextBase/ContextStore/ContextAlgebra/ContextLogic/
  ContextTypeLanguage/ContextBasicDenotation/Denotation/ContextTyping route.

Everything in ContextBase, ContextStore, ContextAlgebra, ContextLogic,
ContextTypeLanguage, ContextBasicDenotation, Denotation, and ContextTyping is
currently closed.


Root proof route
----------------

The current top-level TLet route is intentionally thin:

1. ContextTyping/theories/Soundness.v

   fundamental_let_case
     depends on: fundamental_let_case_direct
     role: public TLet soundness entry.
     similarity/replacement:
       This is a compatibility wrapper.  It can be replaced directly by
       fundamental_let_case_direct if we do not need the public name.

2. ContextTyping/theories/SoundnessDirect.v

   fundamental_let_case_direct
     depends on: denot_tlet_semantic_direct
     role: compile-facing direct route.
     similarity/replacement:
       This is also a thin wrapper.  Keep it only if we want a stable
       SoundnessDirect entry while the rest of the typing induction is restored.

3. ContextTyping/theories/TLetDenotation.v

   denot_tlet_semantic_direct
     depends on: denot_tlet_direct_in_ctx
     role: semantic TLet bridge name for ContextTyping.
     similarity/replacement:
       Thin wrapper.  It can be inlined into SoundnessDirect, but keeping it
       separates "typing case bridge" from "direct denotation call".

4. ContextTyping/theories/TLetDirect.v

   denot_tlet_direct_in_ctx
     depends on:
       - ctx_erasure_under_comma_bind_env_fresh
       - atom_env_to_lty_env_insert
       - atom_env_to_lty_env_closed
       - lty_env_to_atom_env_atom_env
       - tlet_intro_denotation
     role:
       Converts the atom context used by ContextTyping into the lvar context
       consumed by Denotation.TLet.  This is the real bridge from typing to the
       denotation theorem.
     similarity/replacement:
       This lemma is necessary unless the typing judgment is changed to expose
       lty_env directly.  The two atom/lvar conversion rewrites are genuine
       boundary lemmas, not proof duplication.

   denot_tlet_direct_total_model_in_ctx
     depends on:
       - denot_tlet_direct_in_ctx
       - denot_ty_total_model_total
     role:
       Convenience variant when totality is packaged in a total-model premise.
     similarity/replacement:
       Replaceable by denot_tlet_direct_in_ctx plus denot_ty_total_model_total.
       Static scan suggests it is not on the current main TLet route.


Main denotation theorem
-----------------------

5. Denotation/theories/TLet.v

   tlet_intro_denotation
     depends on:
       - tlet_intro_denotation_gas_zero
       - normalize_models_ands / normalize_models_ands_in / normalize_models_ands_goal
       - tlet_support_solver / solve_tlet_sidecond / solve_tlet_guard
       - ty_denote_gas_scope_from_relevant_guard
       - res_models_forall_ext_transport
       - res_models_impl_intro / res_models_impl_elim
       - expr_result_formula_tlete_to_body_ext
       - res_models_open_ty_denote_gas_to_open
       - res_models_open_ty_denote_gas_from_open
       - ty_denote_gas_tapp_tlete_assoc
       - tlet_intro_denotation_wand_case
     role:
       The main gas induction proving that the opened body denotation in the
       extended context implies the let-expression denotation in the base
       context.
     similarity/replacement:
       This is the core theorem and should not be replaced by wrappers.  The
       proof currently relies on two named obligations in TLetSupport.  The
       Over/Under/Arrow cases use a common pattern: split the guard, transport
       the forall witness across the result extension, and then recurse or
       transport the opened denotation.

   tlet_intro_denotation_gas_zero
     depends on:
       - context_ty_wf_formula_models_iff
       - basic_world_formula_models_iff
       - expr_basic_typing_formula_tlete_intro
       - basic_typing_lty_env_to_atom_env_relevant
       - expr_total_formula_tlete_intro_from_result_extension
       - storeA_restrict_dom
       - dom_insert_L
       - formula_scoped_true_iff
     role:
       Base guard transfer for gas 0.
     similarity/replacement:
       This is specific to TLet but its subproofs are exactly the formula-level
       wrappers from ContextBasicDenotation and Denotation.Core.  It should stay
       here unless the guard itself becomes a separate reusable API.

     status: Admitted.
     intended dependencies:

       - expr_result_formula transport through operational equivalence.

     role:
       Converts denotation of tlete e1 (tapp_tm e2 y) into denotation of
       tapp_tm (tlete e1 e2) y at the end of the Arrow/Wand result branch.
     similarity/replacement:
       This is not a wrapper.  It should eventually be proved from a more
       general observational transport lemma for terms.

   tlet_intro_denotation_wand_case
     status: Admitted.
     intended dependencies:
       - the same forall extension transport used by Arrow
       - separating implication/resource-product transport
       - recursive IH

     role:
       Wand-specific branch of tlet_intro_denotation.
     similarity/replacement:

       Static scan suggests these are not on the current main proof route.
       They look like earlier proof-shape documentation and should be reviewed
       after Wand/assoc are closed.  If unused, delete them or turn the useful
       statement into a lower-layer extension lemma.

Static unused or low-use candidates
-----------------------------------

The following are not on the current root route from fundamental_let_case to
tlet_intro_denotation, or are only used by wrappers.  They are candidates for
future cleanup after the two remaining TLetSupport obligations are closed.

ContextTyping:

- denot_tlet_direct_total_model_in_ctx
  Replace with denot_tlet_direct_in_ctx plus denot_ty_total_model_total.

- context_typing_wf_regular_denotation
- context_typing_wf_to_total_model
- entails_total_to_total_model
  These package total-model/regularity facts.  They are not used by the direct
  TLet route.  Keep if they are the intended public API for non-TLet cases;
  otherwise delay exporting them.

- sub_type_under
- ctx_sub_under
- branch_unreachable
  Used by the typing syntax/rules, not by the current direct TLet proof route.
  Keep if the typing system skeleton is considered live.

Denotation:

- ty_denote_gas_saturate
- ty_denote_gas_saturate_ge
  Currently not used by the TLet route.  They are general gas facts and may be
  useful for the public ty_denote wrapper; consider moving them away from hot
  imports if compile time matters.

- ty_denote_gas_extend_typed_extension
- ty_denote_gas_extend_typed_extension_zero
- denot_ty_lvar_guard_extend_typed_extension
  Useful typed-extension weakening facts, but static scan shows they are not
  directly used by current TLet.  Review after Wand is closed.

- tlet_extensions_commute_obligation
- tlet_extensions_commute_widened_obligation
- tlet_arrow_arg_extend_obligation
  Appear to be older proof-plan obligations.  If no future branch uses them,
  delete or replace with general ContextAlgebra extension lemmas.

ContextLogic:

- res_models_forall_map_same_fv
- res_models_forall_congr_same_fv
  Older same-fv transport style.  For current TLet, res_models_forall_ext_transport
  is the stronger/relevant tool.  Keep only for simple congruence proofs.

- res_models_forall_full_world_iff
  Not always used directly, but it is conceptually important: it documents the
  equivalence between the new fv-input FForall and the old full-world intuition.
  Keep for now.

General cleanup rule:

If a lemma's proof is just "apply a more general lemma" and the lemma is not a
stable public boundary name, prefer deleting it and updating call sites to the
general lemma.  If a lemma packages a cross-layer conversion, for example
atom_env_to_lty_env or lty_env_to_atom_env, keep it even when the proof is short.
