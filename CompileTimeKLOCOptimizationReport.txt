Compile-Time KLoC Optimization Report
=====================================

Scope
-----

This pass measured Rocq compile time per thousand source lines for the
non-CoreLang hotspot files.  CoreLang was intentionally left out, following the
current optimization target.

The working threshold was about 5 seconds/KLoC for substantive files.  Small
files can look worse than this because import/loading time dominates their
measured runtime; those are not automatically proof-script hotspots.

Changes Made
------------

- `ContextLogic/theories/FormulaSemantics.v`
  - Replaced two expensive `set_solver` calls in the `FBWand` projection case
    with explicit subset proofs using `formula_open_env_fv_subset`.
  - Result: about 6.4s/KLoC -> about 3.0s/KLoC.

- `ContextTyping/theories/SoundnessApp.v`
  - Replaced one expensive set-search proof in `app_arrow_result_to_target`
    with a direct `fv_tapp_tm`/`elem_of_map` proof.
  - Result: about 14.5s/KLoC -> about 2.6s/KLoC.

- `ContextTyping/theories/Soundness.v`
  - Added local Soundness-set helpers for recurring proof shapes:
    `fv_cty` freshness to `context_ty_lvars` freshness, `tlete` freshness, and
    the singleton-open-env atom shape `{[y]} ∪ ∅`.
  - Replaced repeated Let/LetD/LamD set searches with these local helpers and
    direct fresh facts.
  - Result: about 15.5s/KLoC baseline, then about 6.4s/KLoC after the first
    LamD cleanup, then about 2.9s/KLoC after local helper extraction.

Notes For Future Passes
-----------------------

- Prefer local directory helper lemmas or small Ltac wrappers for recurring
  set-shape obligations in upper-level proofs.  This keeps proof scripts
  readable while avoiding repeated global set search.
- Do not chase high KLoC numbers in small files until `rocq c -time` shows a
  real slow command.  `BasicTypingOpen.v` is currently the clearest example of
  fixed import/loading cost rather than a proof bottleneck.
