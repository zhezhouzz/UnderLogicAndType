Set / Unused Hypothesis Cleanup Analysis
========================================

Scope
-----

This file tracks proof-engineering cleanup opportunities around:

- frequent `set_solver` / `better_set_solver` use;
- local `set (...)` abbreviations that may indicate missing normalization or
  reusable shape lemmas;
- hypotheses introduced by `intros` that appear not to be used.

The unused-hypothesis scan is intentionally conservative: it is a textual
scan, so it can report false positives when hypotheses are used through
`auto`, `eauto`, typeclass search, Ltac pattern matching, notation expansion,
or compatibility wrapper statements.

Existing Infrastructure
-----------------------

`ContextBase/Tactics.v` already provides:

- `set_prune_hyps_safe`
- `set_prune_hyps`
- `better_set_solver!`
- `better_set_solver!!`

These are exactly the "clear irrelevant hypotheses before solver" layer we
want. The right optimization is to route folder-local solvers through these
pruning variants, not to duplicate another global clearing tactic.

Why not delete many premises immediately?
-----------------------------------------

Some apparently unused premises are deliberate:

- compatibility wrappers preserve old lemma statements;
- proof scripts may use assumptions through Ltac or automation;
- some premises encode side conditions useful to callers, even if a specialized
  proof no longer needs them.

The safe policy is:

1. use `_` for locally unused proof variables when the statement should remain;
2. remove theorem premises only when all callers are known and the simplification
   clearly improves the public statement;
3. prefer regular lemmas when the same side fact is repeatedly asserted.

  - extension lemmas report unused freshness/extension witnesses; these are
    public interface lemmas and should not be changed without checking callers.
- `Denotation/theories/ResultFirstOpen.v`
  - some opening helpers report unused freshness facts. These are good
    candidates for a later normalization-interface pass.

Current Residual / Deferred
---------------------------

- `set (...)` in larger proofs often introduces semantic abbreviations
  (`Σrel`, `gas`, `τx`, `m1d`, etc.); these should not be blindly removed.
- `set_solver` in `TypeEquivTLet.v` is still heavy. Some repeated freshness
  blocks likely deserve result-first/TLet-specific regular lemmas.
- The best next cleanup direction is not more global tactic power, but local
  regular lemmas that produce exactly the stdpp facts currently proved by
  repeated `clear -...; set_solver`.
