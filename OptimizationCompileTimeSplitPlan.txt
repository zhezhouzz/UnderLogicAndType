# Compile-Time And File-Splitting Optimization Plan

## Purpose

This optimization pass uses `TIMED=1` and Rocq `-time` data to reduce compile
time and keep large files navigable.  File splitting should follow semantic
boundaries, not arbitrary line counts.

This pass does not delete lemmas.  When a file is split, keep the old public
file as a lightweight re-export/orchestration file unless the user explicitly
asks otherwise.

## Skill Guidance Used

- Use 50 second focused checks.
- If a proof times out or progress is unclear, use `-time`.
- Stop after four timing/debugging rounds and reassess.
- Split oversized files or decompose lemmas instead of turning timing into the
  proof strategy.

## Compile-Time Optimization Workflow

1. Measure before changing.

   ```sh
   timeout 180s make TIMED=1
   ```

2. For a slow file, run focused timing.

   ```sh
   timeout 50s make TIMED=1 Denotation/theories/TypePersist.vo
   ```

3. If a specific proof is slow, run the underlying Rocq command with `-time`
   or inspect generated make output to find the slow sentence.

4. Replace broad solver calls only after identifying the slow shape.
   - Add a small lemma if the same side condition appears twice.
   - Add a local deterministic tactic only for normalized goals.

5. Re-measure and record before/after in this file or a follow-up status
   section.
