# Normalize-First Optimization Plan

## Purpose

This optimization pass makes proofs normalize the semantic/syntax boundary
early, before extracting helper lemmas.  The goal is to avoid helper statements
that expose raw `formula_open`, `lty_env_open_one`, `typed_lty_env_bind`,
`relevant_env`, erased syntax, locally-nameless open/shift, or depth noise.

This pass does not delete lemmas.  Existing lemmas should be preserved as
wrappers when they are already used by downstream proofs.

## Skill Guidance Used

- Normalize at theorem and Fundamental-case boundaries.
- Normalization tactics should preserve equality.
- Side-condition solving belongs in dedicated solvers or existing `better_*`
  solvers, not in equality normalization tactics.
- If a helper premise contains raw open/relevant-env syntax noise, normalize
  earlier.

## Concrete Optimization Work

1. Add or strengthen equality-only normalizers.
   - `ty_denote_open_singleton_norm`
   - `ty_denote_open_two_slots_norm`
   - `result_first_arrow_open_norm`
   - `result_first_wand_open_norm`
   - `persist_ret_value_open_norm`
   - `over_open_body_norm`
   - `under_open_body_norm`

2. Keep side-condition tactics separate.
   - `denotation_open_side_solve`
   - `soundness_open_side_solve`
   - `persist_open_side_solve`

3. Replace proof-local `change`, `replace`, and repeated rewrite chains with
   one named normalization lemma whenever the same raw shape occurs twice.

4. Normalize helper statements before extraction.
   - New helper statements should mention `over_open_body φ z` rather than
     `formula_open 0 z (...)`.
   - New helper statements should mention inserted free-variable environments
     rather than `lty_env_open_one 0 z (typed_lty_env_bind ...)`.
   - New helper statements should mention simplified `erase_ty` results.

5. Preserve old lemmas.
   - If an existing lemma has a noisy statement but is used externally, keep it
     as a wrapper that immediately calls the normalized lemma.
   - Mark wrappers as compatibility in comments, but do not delete them in this
     optimization pass.

## Suggested Order

1. Start with `ResultFirstOpen.v` because it is shared by Denotation and
   Soundness.
2. Add normalized `CTPersist` ret-value lemmas in `TypePersist.v`.
3. Replace local normalization blocks in `SoundnessPersist.v`.
4. Replace repeated result-first normalization in `SoundnessLam*`,
   `SoundnessApp*`, and `SoundnessFixApply.v`.
5. Re-run the raw-shape scan:

   ```sh
   rg -n "formula_open 0 .*ty_denote_gas|lty_env_open_one 0 .*typed_lty_env_bind|cty_open 0 .*cty_shift 0|cbn \\[cty_depth\\]" \
     Denotation ContextTyping
   ```
