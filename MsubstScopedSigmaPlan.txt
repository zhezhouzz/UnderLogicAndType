Msubst scoped-sigma proof plan
================================

Goal
----

Prove the Soundness-facing msubst transport while keeping the public theorem
simple: callers pass the full base projection store, but the recursive
denotation proof only receives the part of the store that is relevant to the
current formula.

Public boundary
---------------

The public theorem keeps the existing shape:

  ty_denote_gas_msubst_store_from_typing_wf_iff

It accepts context_typing_wf_erased, storeA_has_type for the base store, and
base_store_projection.  Before calling the internal theorem, it defines:

  p := ty_denote_gas gas Σ τ e
  σp := store_restrict σ (formula_fv p)

Then it rewrites formula_msubst_store σ p to formula_msubst_store σp p and
proves that instantiating e with σp is equal to instantiating e with σ.

Internal invariant
------------------

The internal scoped IH explicitly requires the store to be formula-relevant:

  let p := ty_denote_gas gas Σ τ e in
  dom (σ : StoreT) ⊆ formula_fv p ->
  ...

Every recursive call must pass a restricted store:

  p_child := ty_denote_gas gas Σchild τchild echild
  σ_child := store_restrict σ (formula_fv p_child)

This keeps atom_store_has_ltype and store_singleton_projection local to the
formula being proved, instead of requiring child worlds to carry unrelated
base variables.

Execution order
---------------

1. Split TypeDenoteMsubst.v into:
   - TypeDenoteMsubstCore.v
   - TypeDenoteMsubstGuard.v
   - TypeDenoteMsubstKeepDomain.v
   - TypeDenoteMsubstCompat.v
   - TypeDenoteMsubstScoped.v
   with TypeDenoteMsubst.v as an export aggregator.

2. Update the scoped IH statement and thread the dom-subset premise through:
   - ty_denote_gas_msubst_store_local_scoped_succ
   - ty_denote_gas_msubst_store_local_scoped_iff
   - ty_denote_gas_msubst_store_local_iff
   - ty_denote_gas_msubst_store_from_typing_wf_iff

3. Update structural recursive cases:
   - Inter, Union, Sum use σ1/σ2 restricted to each branch formula_fv.
   - Over/Under keep using keep-domain helpers, with restricted σ where useful.
   - Arrow/Wand define σ_arg and σ_res from p_arg and p_res, then call the IH
     backward for the argument and forward for the result.

4. Reuse existing helpers wherever possible:
   - formula_msubst_store_restrict_fv_subset
   - res_models_formula_msubst_store_agree_fv
   - store_singleton_projection_restrict
   - atom_store_has_ltype_restrict_lvars
   - lstore_instantiate_tm_lift_free_agree_subset
   - formula_fv_ty_denote_gas_subset_relevant_pre_open

5. Compile split files independently.  If any file exceeds 80 seconds, use
   rocq c -time on that file and split or factor the slow proof before
   continuing.

Completion criteria
-------------------

- TypeDenoteMsubst.v is a thin aggregator.
- The scoped IH carries dom σ ⊆ formula_fv p.
- The public theorem does not expose this internal premise.
- The new split files compile incrementally.
- Remaining admits, if any, are named Arrow/Wand or old compatibility
  obligations, not anonymous admits.

Wand-first route
================

Priority
--------

Prove ty_denote_gas_msubst_store_wand_local_scoped_succ before Arrow.
Wand is the higher-risk case because recursive calls happen under a frame
world and a product world; both must use σ restricted to the child formula FV.

Logic helper
------------

Add res_models_forall_full_world_impl_wand_map in FormulaConnectives.  It
transports formulas of the shape:

  FForall (FImpl A1 (FWand B1 C1))
  -> FForall (FImpl A2 (FWand B2 C2))

by reusing res_models_forall_full_world_map and res_models_impl_wand_map.  The
caller supplies opened transports A2 -> A1, B2 -> B1, and C1 -> C2.

Singleton helpers
-----------------

Do not try to prove a Wand frame helper from product-le.  The frame side only
has world_compat n m, not res_product n m <= m.  Instead use the algebraic
compatibility overlap fact: if n is compatible with m, stores in n and stores
in m agree on their overlap.  Combined with store_singleton_projection σ m and
formula scope in n, this gives singleton projection for
store_restrict σ (formula_fv child) in n.

For the consequent world res_product n m Hc, use the existing right-subresource
relation res_product_le_r plus store_singleton_projection_subset_world, because
m <= res_product n m Hc and the child formula scope supplies the needed world
coverage.

Wand proof shape
----------------

1. Split guard/body and prove guard with denot_guard_msubst_store_local_singleton_iff.
2. Use res_models_forall_full_world_impl_wand_map on the body.
3. For the opened Wand antecedent, call denot_msubst_local_scoped_IH_backward
   with σ_arg := store_restrict σ (formula_fv p_arg).
4. For the opened Wand consequent, call denot_msubst_local_scoped_IH_forward
   with σ_res := store_restrict σ (formula_fv p_res).
5. Reuse formula_open_ty_denote_gas_singleton, formula_open_msubst_store_fresh,
   lstore_instantiate_tm_shift_lift_free, and expr_result_formula_msubst_store_open_shift_models_back.

Counterexample note
-------------------

Passing full σ into the Wand frame world is false in general: the frame world
need not contain every atom in dom σ.  The correct invariant is restricted to
formula_fv of the child formula.

Wand blocker discovered during implementation
=============================================

The Logic transport and singleton helpers compiled, but the Wand body still
needs one more general relevant-environment agreement bridge before the scoped
IH can be applied.

After normalizing the Wand body, the forward source body has recursive children
of the form:

  formula_msubst_store σ
    (ty_denote_gas gas
      (typed_lty_env_bind
        (relevant_env Σ (CTWand τx τr) e)
        (erase_ty τx))
      ...)

while the target body has recursive children of the form:

  ty_denote_gas gas
    (typed_lty_env_bind
      (relevant_env Σ (CTWand τx τr)
        (lstore_instantiate_tm (lstore_lift_free σ) e))
      (erase_ty τx))
    ...

The scoped IH can transport term instantiation while keeping the same Σ, but it
cannot by itself change the source child environment to the target child
environment.  Therefore Wand (and probably Arrow) needs a reusable env-agree
transport lemma before applying the IH.

Suggested helper shape
----------------------

For a child type/term pair [τc/ec] that appears under the opened Wand/Arrow body,
prove a general agreement lemma:

  lty_env_restrict_lvars
    (typed_lty_env_bind
      (relevant_env Σ (CTWand τx τr) e)
      (erase_ty τx))
    (relevant_lvars τc ec)
  =
  lty_env_restrict_lvars
    (typed_lty_env_bind
      (relevant_env Σ (CTWand τx τr)
        (lstore_instantiate_tm (lstore_lift_free σ) e))
      (erase_ty τx))
    (relevant_lvars τc ec)

under premises saying:

  relevant_lvars τc ec
    ⊆ {[LVBound 0]} ∪ context_ty_lvars τx ∪ context_ty_lvars τr ∪
       tm_lvars (lstore_instantiate_tm (lstore_lift_free σ) e)

and the corresponding source-side coverage after msubst/restrict.  A cleaner
variant may state this as agreement-on [relevant_lvars τc ec] rather than
literal equality, then use res_models_ty_denote_gas_env_agree_on.

Existing lemmas likely reusable:

  - res_models_ty_denote_gas_env_agree_on
  - arrow_body_relevant_env_agree_from_basic_context_ty
  - wand_body_relevant_env_agree_from_basic_context_ty
  - lstore_instantiate_tm_lift_free_agree_subset
  - tm_lvars_lstore_instantiate_lift_free_closed_subset
  - formula_fv_ty_denote_gas_subset_relevant_pre_open

Implementation order adjustment
-------------------------------

1. Factor this as a general RelevantEnv lemma, not a Wand-specific proof script.
2. Use it in Wand antecedent/result before calling the scoped IH.
3. Reuse the same lemma for Arrow.

Relevant-env aligned implementation update
==========================================

The least disruptive implementation is not to replace the already-working
single-Σ scoped IH definition.  Instead, keep that IH as the core induction
principle and add two thin wrappers at the recursive-call boundary:

  - denot_msubst_local_scoped_IH_forward_env_agree
  - denot_msubst_local_scoped_IH_backward_env_agree

Each wrapper rewrites the source denotation environment to a middle environment
and rewrites the target denotation environment from the middle environment to
the desired target environment, using the formula-level equality
ty_denote_gas_env_agree_on.  Then it calls the existing scoped IH wrapper.

This implements the relevant-env aligned route while preserving the existing
Inter/Union/Sum/Over/Under proofs.  Arrow/Wand only need to provide the two
environment equalities at the opened argument/result children.  The new
RelevantEnv helpers are:

  - relevant_env_idemp
  - relevant_env_restrict_subset
  - arrow_arg_relevant_env_agree
  - wand_arg_relevant_env_agree

Next proof step: use these wrappers inside
ty_denote_gas_msubst_store_wand_local_scoped_succ first, then Arrow.

Specialized body-helper implementation update
=============================================

The top-level Arrow/Wand succ cases are now split successfully:

  - ty_denote_gas_msubst_store_arrow_local_scoped_succ
  - ty_denote_gas_msubst_store_wand_local_scoped_succ

They only split guard/body, use denot_guard_msubst_store_local_singleton_iff
for the guard, and delegate body transport to specialized body helpers.
This keeps the main recursive cases small and compiled.

The remaining active obligations are now:

  - ty_denote_gas_msubst_store_wand_body_scoped
  - ty_denote_gas_msubst_store_arrow_body_scoped

The next implementation should prove Wand body first by introducing smaller
opened-child helpers rather than expanding everything in the body proof.

Minimal Wand opened-child helper shape:

1. Opened basic-world guard transport:

   formula_open 0 y (formula_msubst_store σ
     (basic_world_formula (<[LVBound 0 := erase_ty τx]> ∅)))
   is model-equivalent to
   formula_open 0 y
     (basic_world_formula (<[LVBound 0 := erase_ty τx]> ∅))

   under y ∉ dom σ.  This should be a small formula-open/msubst freshness
   proof.

2. Opened Wand argument transport:

   In a frame world n compatible with the opened forall world my, transport
   the target opened argument back to the source msubsted opened argument.
   The proof should:

   - normalize formula_open and formula_msubst_store;
   - use res_models_open_ty_denote_gas_to_open;
   - align source/middle/target relevant envs with wand_arg_relevant_env_agree
     and denot_msubst_local_scoped_IH_backward_env_agree;
   - use store_singleton_projection_compat_restrict_of_scope for the restricted
     σ needed by the frame world;
   - return with res_models_open_ty_denote_gas_from_open when necessary.

3. Opened Wand result transport:

   In res_product n my Hc, transport the source msubsted opened result to the
   target opened result.  The proof should:

   - normalize the opened tapp body with open_tapp_tm_shift_bvar0_lc or the
     existing tapp/open lemma;
   - align relevant envs with
     wand_body_relevant_env_agree_from_basic_context_ty;
   - use denot_msubst_local_scoped_IH_forward_env_agree;
   - use store_singleton_projection_product_restrict_of_scope for the
     restricted σ in the product world.

Arrow body should then reuse the same pattern with impl2 instead of wand:

  - argument uses denot_msubst_local_scoped_IH_backward_env_agree;
  - result uses denot_msubst_local_scoped_IH_forward_env_agree;
  - singleton projection is obtained from
    store_singleton_projection_forall_restrict_of_scope, since there is no
    separate frame/product world.

Risk note:

The hard part is not the FForall/FImpl/FWand structure anymore.  The remaining
risk is exact alignment of opened formula shapes after formula_open,
formula_msubst_store, and ty_denote_gas open transport.  If this remains
too brittle, the next fallback should be to make the opened-child helpers'
statements match the exact formula_open goals produced by
res_models_forall_full_world_impl_wand_map, rather than trying to normalize
them to a prettier shape.

Wand opened-helper correction
-----------------------------

The first iff-shaped opened argument helper was too weak: it had no singleton
projection premise for the frame world.  That is not semantically valid in
general, since the frame world need not contain all of dom σ.  The correct
shape is directional:

  - target -> source-msubst uses a restricted singleton projection obtained
    from the target model's scope;
  - source-msubst -> target uses a restricted singleton projection obtained
    from the source model's scope.

The same applies to the result helper in the product world.  The body proof
should pass these projection facts from:

  - store_singleton_projection_compat_restrict_of_scope for the frame argument;
  - store_singleton_projection_product_restrict_of_scope for the product result.

The directional helper statements should keep the formula_open shape produced
by the Logic transport.  They can normalize internally to opened denotations
and then call the env-agree IH wrappers.

Wand reverse completed-frame route
----------------------------------

The reverse Wand direction cannot require the target argument to be scoped in
the original frame world n.  Instead, after receiving

  n ⊨ formula_open 0 y (formula_msubst_store σ source_arg)

we complete the frame with the singleton part fixed by σ for the opened target
argument.  The framework is:

  - build a singleton frame from store_restrict σ X, where X contains the
    opened source/target argument FV needed by the child IH;
  - prove this singleton frame is compatible with n by using the parent
    projection store_singleton_projection σ my and world_compat n my;
  - let nσ be the product of n and this singleton frame;
  - lift the source-msubsted argument model from n to nσ by minimality/agreement;
  - prove target argument scope in nσ using the source-msubsted scope plus the
    added singleton part;
  - use denot_msubst_wand_opened_arg_source_to_target on nσ;
  - run the target Wand on nσ;
  - transport the result back to source-msubst and drop the completed frame by
    an agreement/minimality lemma for the source-msubsted result.

The active named helpers for this route are:

  - denot_msubst_wand_completed_arg_frame
  - denot_msubst_wand_drop_completed_frame_result

The first reusable low-level lemma to prove is:

  world_compat_singleton_restrict_from_projection

which says that if n is compatible with m, and m projects to σ on dom σ, then n
is compatible with singleton_world (store_restrict σ X).  This avoids
re-proving the overlap argument inside the Wand proof.
