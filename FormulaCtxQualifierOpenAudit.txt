# Formula / Context Type / Qualifier Open Audit

## Purpose

Track proof-facing residual opens after the relevant-env normalization pass.
This audit is intentionally broader than `OpenRelevantNormalizationAudit.txt`:
it includes `formula_open`, `cty_open`, `qual_open`, and env/context open
forms.  Not every occurrence is bad.  Dependent type results such as
`cty_open 0 x τ` are semantically meaningful and should remain; raw opens
that are only boundary-normalization artifacts should be removed at their
source.

## Classification

- **Must normalize at source**
  - Helper statements in `Denotation` / `ContextTyping` with raw
    `formula_open ... ty_denote_gas`, result-first value-body opens, or
    `lty_env_open_one ... typed_lty_env_bind`.
  - Proof-local asserts that recreate raw opened denotations only to rewrite
    them immediately.

- **Usually legitimate dependent open**
  - `cty_open 0 x τ` when `x` is a real program argument / binder and the
    result type is dependent on it.
  - `qual_open_atom ...` in normalized body abbreviations such as
    `over_open_body` / `under_open_body`.

- **Support / normalizer internals**
  - Files defining syntax/open lemmas, e.g. `TypeDenote.v`,
    `RelevantEnv.v`, `BasicTypingOpen.v`, `TermOpen.v`, `Qualifier.v`.
  - These may contain raw opens, but public helper statements should still
    expose normalized forms whenever possible.

## Iteration Queue

1. **SoundnessAppD formula-open statements**
   - Inspect raw `formula_open 0 x` / double-open helper statements.
   - Normalize at source if they are just opened argument/result boundary
     facts.
   - Preserve `cty_open 0 x τ` where it is the actual dependent result type.

2. **SoundnessLamArrow / SoundnessLamWand result-first opens**
   - Same approach as `SoundnessApp`: normalize outer function-result value
     formula at the helper that first opens it.

3. **TypeEquivFiberTransport**
   - Classify duplicate old transport helpers against `TypeEquivArrow.v` and
     `TypeEquivWand.v`.
   - Normalize canonical helpers or remove unused duplicate wrappers.

4. **TypePersistArrow / TypePersistWandReverse**
   - Replace proof-local raw result-first value opens with normalized
     `arrow_value_denote_gas_with` / `wand_value_denote_gas_with` statements.

5. **Support internals**
   - Leave raw opens inside normalizer proofs unless a public statement leaks
     them.
