# Long Proof Pattern Refactor Analysis

Scope: `Denotation/theories`, `ContextTyping/theories`, and
`ContextBasicDenotation/theories`.

Goal: identify proofs over 200 lines, analyze repeated proof patterns, and
propose reusable lower-level lemmas or tactic improvements.  I also inspected
120-200 line proofs because they often share the same shape and are useful
when designing common helpers.

## Executive Summary

There are only four proofs longer than 200 lines in the current upper stack:

- `ContextBasicDenotation/theories/BasicTypingFormula.v`
  - `basic_has_ltype_msubst_store_back_by_derivation`, 331 proof lines.
- `Denotation/theories/TypeDenote.v`
  - `formula_open_env_ty_denote_gas`, 288 proof lines.
- `Denotation/theories/TypeDenoteOpen.v`
  - `res_models_open_ty_denote_gas_iff`, 269 proof lines.
- `Denotation/theories/TypeEquivCore.v`
  - `tm_equiv_tlete_body_extension`, 204 proof lines.

The repeated patterns are not random.  They fall into five reusable families:

1. **Induction over syntax/typing with repeated binder cases.**
   Examples: `basic_has_ltype_msubst_store_back_by_derivation`,
   `lstore_instantiate_open_back_mutual`,
   `lstore_instantiate_lift_free_lvars_at_mutual`.

2. **Induction over `gas`/`context_ty` with repeated guard/body splitting.**
   Examples: `formula_open_env_ty_denote_gas`,
   `res_models_open_ty_denote_gas_iff`,
   `ty_denote_gas_tm_result_equiv`.

3. **Open-env / open-one normalization under arrow/wand binders.**
   Examples: `formula_open_env_ty_denote_gas`,
   `res_models_open_arrow_body_iff`,
   `res_models_open_wand_body_iff`,
   `wand_*_relevant_env_agree_open_one_core`.

4. **Resource/result-extension transport for TLet.**
   Examples: `tm_equiv_tlete_body_extension`,
   `tlete_total_of_result_ext`,
   `fundamental_let_case`.

5. **Basic-world / result-alias typing reconstruction.**
   Examples: `basic_world_formula_result_alias_target`,
   `result_ext_typed_of_guard`,
   `basic_world_formula_extend_by_typed_extension`,
   `basic_world_formula_drop_result_extension`.

The best next refactor is not to make a single giant tactic.  The long proofs
need 6-8 small semantic helper lemmas plus two localized normalization Ltacs.
The likely highest-return targets are:

- a binder-open packet lemma for `lstore_instantiate_*_at` and
  `lty_env_msubst_store_insert_fresh`;
- an open-env body-normalization lemma for arrow/wand denotation bodies;
- a guard-splitting and guard-rebuilding helper for `ty_denote_gas`;
- a result-extension store packet for TLet/result alias;
- a relevant-env open-one lookup/agreement helper parameterized by the body
  support lemma.

## Detailed Analysis by Lemma

### 1. `basic_has_ltype_msubst_store_back_by_derivation`

File: `ContextBasicDenotation/theories/BasicTypingFormula.v`.

Current role:

- It proves the back direction of msubst typing by induction on
  `basic_has_ltype`.
- It is only used to derive the wrapper lemmas
  `basic_value_has_ltype_msubst_store_back_at` and
  `basic_tm_has_ltype_msubst_store_back_at`.
- It is still semantically useful.  It is not old fiberwise-only code.

Repeated patterns:

- Repeated destruct of instantiated values:
  `destruct v as [c|x|n|s e|Tf vf]`.
- Repeated free-var lookup case:
  rewrite `lstore_free_part_lift_free`;
  destruct `σ !! x`;
  if `Some`, recover typing via
  `basic_value_has_ltype_msubst_store_back_fvar`;
  if `None`, use inversion/discriminate.
- Repeated bound-var impossible case:
  expand `lstore_bound_part`, rewrite
  `lstore_lift_free_lookup_bound`, then discriminate or reconstruct
  `BVT_BVar`.
- Repeated binder case:
  choose fresh `y`;
  build `atom_store_has_ltype_insert_fresh`;
  rewrite `lty_env_msubst_store_insert_fresh`;
  rewrite open/instantiate commuting lemma;
  call IH.

Recommended extraction:

1. Add a local Ltac or named helper for value-shape inversion:

   ```coq
   Ltac invert_lift_free_instantiated_value :=
     cbn [lstore_instantiate_value_at
          lstore_instantiate_value_split_at] in *;
     try rewrite lstore_free_part_lift_free in *;
     repeat match goal with
     | H : context [lstore_bound_part (lstore_lift_free _)] |- _ =>
         rewrite StoreInterface.lstore_bound_part_lookup in H;
         rewrite lstore_lift_free_lookup_bound in H
     end.
   ```

   This should live near `BasicTypingFormula.v`, not globally, because it is
   proof-engineering for one mutual induction shape.

2. Extract the binder packet:

   ```coq
   Lemma basic_msubst_back_binder_packet Σ σ y T :
     y ∉ dom (σ : gmap atom value) ->
     atom_store_has_ltype Σ σ ->
     atom_store_has_ltype (<[LVFree y := T]> Σ) σ.
   ```

   A stronger exact version already exists in spirit as
   `atom_store_has_ltype_insert_fresh`.  The new wrapper should package the
   freshness side in the shape this proof repeatedly needs.

3. Extract the open/instantiate rewrite packet:

   ```coq
   Lemma lstore_instantiate_tm_at_open0_fresh_lift_free_from_typed_store ...
   Lemma lstore_instantiate_value_at_open0_fresh_lift_free_from_typed_store ...
   ```

   These would combine:
   `atom_store_has_ltype_closed`,
   `lstore_instantiate_*_at_open0_fresh_lift_free`,
   and the recurring `y ∉ dom σ ∪ fv_*` side conditions.

4. Keep the mutual induction itself.  Do not try to replace it with
   automation.  The proof is long because the typing relation is structural;
   the right cleanup is to make each constructor branch small.

Estimated payoff:

- 331 lines could plausibly drop to 180-220 lines.
- More importantly, the proof would become robust to future syntax changes.

Risk:

- Low semantic risk.
- Medium proof-engineering risk because destructing instantiated values is
  brittle.  Work incrementally.

### 2. `formula_open_env_ty_denote_gas`

File: `Denotation/theories/TypeDenote.v`.

Current role:

- Main formula-level theorem for opening an entire denotation formula by an
  open environment.
- Used by `TypeDenoteOpen.v` for singleton open models.

Repeated patterns:

- Every non-base type case starts with:

  ```coq
  cbn [ty_denote_gas].
  rewrite formula_open_env_and.
  rewrite formula_open_env_denot_guard by ...
  rewrite open_cty_env_* by ...
  cbn [ty_denote_gas].
  ```

- `CTInter/CTUnion/CTSum` are identical modulo connective:
  `FAnd`, `FOr`, `FPlus`, and `open_cty_env_*`.
- `CTArrow` and `CTWand` are identical modulo
  `open_cty_env_arrow/wand` and `formula_open_env_impl/wand`.
- `CTOver` and `CTUnder` are identical modulo `over/under`.
- Freshness construction repeats:
  - `Hfreshφ` for qualifier vars after lifting.
  - `Hfreshe` for term vars.
  - `HfreshΣg`, `Hfresh_arg`, `Hfresh_body` for arrow/wand.

Existing helpers already useful:

- `open_env_lift_fresh_for_bound0_bind_dom`.
- `lvars_at_depth_arrow_arg_lift_support_subset`.
- `arrow_body_lvars_lift_support_subset`.
- `lvars_at_depth_wand_arg_lift_support_subset`.
- `lvars_at_depth_wand_body_lift_support_subset`.
- `relevant_env_open_env`.
- `typed_lty_env_bind_open_env_lift`.

Recommended extraction:

1. Add connectives normalization lemmas for recursive denotation branches:

   ```coq
   Lemma formula_open_env_ty_denote_gas_binary_and ...
   Lemma formula_open_env_ty_denote_gas_binary_or ...
   Lemma formula_open_env_ty_denote_gas_binary_plus ...
   ```

   But this may create artificial wrappers.  Better: define a local Ltac:

   ```coq
   Ltac open_env_denot_guard_prefix :=
     cbn [ty_denote_gas];
     rewrite formula_open_env_and;
     rewrite formula_open_env_denot_guard by eauto;
     cbn [ty_denote_gas].
   ```

2. Extract Over/Under body normalization:

   ```coq
   Lemma formula_open_env_over_under_body
   ```

   More concretely:

   ```coq
   Lemma formula_open_env_over_body_lifted η b φ e :
     open_env_fresh_for_lvars (kmap S η) (qual_vars φ) ->
     open_env_fresh_for_lvars η (tm_lvars e) ->
     open_env_values_inj η ->
     formula_open_env η <over-body> = <opened-over-body>.
   ```

   Then Under is a one-line call over the same helper if the connective is a
   parameter, or a twin lemma if parameterization is awkward.

3. Extract Arrow/Wand body mopen normalization:

   ```coq
   Lemma formula_open_env_arrow_wand_body
   ```

   It should package:
   - lifted fresh env for argument/result;
   - open of typed bind;
   - relevant env open;
   - shift/open simplification for `tret bvar0` and `tapp_tm`.

   This is high value because the same shape appears later in
   `res_models_open_arrow_body_iff` and `res_models_open_wand_body_iff`.

Estimated payoff:

- Could reduce this proof by 80-120 lines.
- More importantly, future Arrow/Wand denotation changes would touch one
  helper instead of two large branches.

Risk:

- Low semantic risk.
- Medium proof-size risk if the extracted body lemma has a giant statement.
  Prefer local Ltac plus 2 body-specific lemmas over one over-general lemma.

### 3. `res_models_open_ty_denote_gas_iff`

File: `Denotation/theories/TypeDenoteOpen.v`.

Current role:

- Models-level singleton open theorem for type denotation.
- This is a true semantic bridge and should be retained.

Repeated patterns:

- Every case:
  - rewrite `formula_open_and`;
  - split guard/body using `res_models_and_iff`;
  - apply `res_models_open_denot_ty_lvar_guard_iff`;
  - rewrite `relevant_env_open_one`;
  - transport the body.

- `CTUnion` and `CTSum` duplicate guard scope extraction:

  ```coq
  assert (HguardF : m ⊨ FAnd ...).
  { repeat rewrite res_models_and_iff. exact Hguard. }
  ```

- The two directions of `CTUnion` have nearly identical scope proof chunks
  using:
  - `ty_denote_gas_scope_of_guard`
  - `formula_fv_open_ty_denote_gas_subset_relevant`
  - `denot_guard_term_type_fv_scope`.

- `CTArrow` and `CTWand` are already delegated to
  `res_models_open_arrow_body_iff` and `res_models_open_wand_body_iff`.
  This is a good pattern.

Existing helpers already useful:

- `res_models_open_arrow_body_iff`.
- `res_models_open_wand_body_iff`.
- `res_models_or_transport_between_worlds`.
- `res_models_plus_map`.
- `ty_denote_gas_scope_of_guard`.
- `denot_guard_term_type_fv_scope`.

Recommended extraction:

1. Add guard packet accessor:

   ```coq
   Lemma denot_guard_opened_formula_scope
       k y Σ τ e m :
     m ⊨ <opened guard formula> ->
     formula_fv (ty_denote_gas gas ... τi ...) ⊆ world_dom m.
   ```

   Or more simply:

   ```coq
   Ltac denot_guard_packet Hguard :=
     let HguardF := fresh "HguardF" in
     assert (HguardF : _)
       by (repeat rewrite res_models_and_iff; exact Hguard).
   ```

   Since the formula is huge, a Ltac packet may be cleaner than a named lemma.

2. Extract `res_models_open_union_branch_transport`:

   A short lemma wrapping the `res_models_or_transport_between_worlds` call
   and both branch FV side conditions.

3. Extract `res_models_open_sum_branch_transport`:

   Similar wrapper around `res_models_plus_map`.

4. Keep Arrow/Wand delegation as-is.  This proof is already improved there.

Estimated payoff:

- 269 lines could drop to about 170-190 lines.
- Most readability gain is in `Union`/`Sum` branches.

Risk:

- Low semantic risk.
- Medium proof ergonomics risk because named formulas become very large.
  If helper statements get unreadable, prefer local Ltac packets.

### 4. `tm_equiv_tlete_body_extension`

File: `Denotation/theories/TypeEquivCore.v`.

Current role:

- Core operational equivalence that enables the new compact
  `tlet_intro_denotation`.
- This lemma is important and should be preserved.

Repeated patterns:

- Build a scope for the opened body from the whole let:
  `Hscope_body`.
- Derive output binding `σ !! x = Some vx` from the extension:
  `Hstore_result`.
- Compare inserted/restricted stores for a fresh alias:
  `Hinsert_restrict_y`.
- Forward and backward directions are symmetric:
  - choose fresh `y`;
  - eliminate/evaluate `tlete`;
  - recover `vx`;
  - use `tm_eval_in_store_open_alias`;
  - use `tm_eval_in_store_agree_on_fv`.

Existing helpers already useful:

- `expr_result_extension_apply_total_iff`.
- `resA_extend_by_store_iff`.
- `tm_eval_in_store_tlete_elim_closed_on`.
- `tm_eval_in_store_tlete_intro_closed_on`.
- `tm_eval_in_store_open_alias`.
- `store_insert_restrict_agree_on_open_fv`.
- `tm_eval_in_store_agree_on_fv`.

Recommended extraction:

1. Extract result-store lookup from extension:

   ```coq
   Lemma result_extension_store_lookup_output
       e x Fx m mx σ vx :
     expr_result_extension_witness e x Fx ->
     res_extend_by m Fx mx ->
     worldA_stores (mx : WorldT) σ ->
     tm_eval_in_store (store_restrict σ (fv_tm e)) e vx ->
     σ !! x = Some vx.
   ```

   This is currently `Hstore_result`.  It is independently useful for other
   let/result-extension proofs.

2. Extract opened body scope:

   ```coq
   Lemma tlete_body_open_scope_from_total
       e1 e2 x Fx m mx σ :
     ...
     tm_lvars (e2 ^^ x) ⊆ lvars_of_atoms (dom σ).
   ```

   This is currently `Hscope_body`.  It belongs in `TermTLet.v` or
   `TermEval.v`, not in denotation saturate core.

3. Extract fresh alias evaluation transport:

   ```coq
   Lemma expr_eval_tlete_body_alias_equiv_store
       e1 e2 x y σ vx v :
     σ !! x = Some vx ->
     y ∉ dom σ ->
     x ∉ fv_tm e2 ->
     y ∉ fv_tm e2 ->
     tm_lvars (e2 ^^ x) ⊆ lvars_of_atoms (dom σ) ->
     tm_eval_in_store σ (e2 ^^ x) v <->
     tm_eval_in_store (<[y := vx]> σ) (e2 ^^ y) v.
   ```

   This is just `tm_eval_in_store_open_alias`, but a version whose
   statement is exactly the TLet use would avoid repeating the same packet.

4. Keep the final theorem itself as the semantic boundary.  It is now used by
   `tlet_intro_denotation`.

Estimated payoff:

- 204 lines could drop to about 100-130 lines.
- The extracted `result_extension_store_lookup_output` would likely help
  future `letd` or result-extension proofs.

Risk:

- Low semantic risk.
- Medium proof dependency risk: these helpers must live below
  `SaturateCore.v`, probably in `ContextBasicDenotation/TermTLet.v` or
  `TermExtension.v`, otherwise cyclic imports appear.

## Near-Miss Proofs and Shared Patterns

### `wand_body_relevant_env_agree_open_one_core`

File: `ContextBasicDenotation/theories/RelevantEnv.v`.

This proof is 173 lines and is a high-value cleanup target.  It is used by:

- `Denotation/theories/TypeEquivWand.v`
- `arrow_body_relevant_env_agree_open_one_core` as a direct wrapper.

Pattern:

- Performs lookup equality after `lty_env_open_one`.
- Repeatedly proves that if the opened variable is in the relevant set then
  the original variable is in source relevant vars.
- Splits cases on `logic_var_open 0 y v`:
  bound 0, bound S, free y, free z.

Recommended lower-level lemmas:

```coq
Lemma logic_var_open_one_inv_bound_succ ...
Lemma logic_var_open_one_inv_free ...
Lemma relevant_lvars_open_body_back
```

The key missing abstraction is not map lookup; it is the support back-transport:

```coq
Lemma opened_body_relevant_lvars_without_y_subset_source
    y τr e_body e_src :
  y ∉ fv_cty τr ->
  tm_lvars e_body ∖ {[LVFree y]} ⊆ lvars_shift_from 0 (tm_lvars e_src) ->
  relevant_lvars (cty_open 0 y τr) e_body ∖ {[LVFree y]} ⊆
  lvars_shift_from 0 (relevant_lvars τr e_src).
```

If this support lemma exists, the map equality proof becomes a short
`storeA_restrict_eq_mono`.

### `wand_arg_relevant_env_agree_open_one_core`

Lemma opened_arg_relevant_lvars_subset_source

### `basic_world_formula_result_alias_target`

File: `Denotation/theories/TypeEquivCore.v`.

This 123-line proof reconstructs `basic_world_formula` for the target
`tret (vfvar x)` after a result alias.

Pattern:

- Scope proof splits target relevant lvars into old type vars or singleton x.
- Store typing proof splits target env lookup into:
  old relevant vars handled by source world;
  x handled by result formula and evaluation typing.

Recommended extraction:

```coq
Lemma relevant_env_ret_fvar_lookup_split
Lemma basic_world_formula_ret_fvar_alias_from_source
```

The first should be a pure RelevantEnv/Store lemma that says every lookup in
`relevant_env Σ τ (tret (vfvar x))` either belongs to
`relevant_env Σ τ e` or is exactly `LVFree x`.

This would remove most of the manual set/store splitting in the proof.

## Solver / Tactic Opportunities

### 1. Better denotation FV solver

Current expensive proof spots are in
`ty_denote_gas_lvars_subset`, especially binary and
Arrow/Wand cases.  The timing pass showed several `set_solver` calls around
0.5-0.8 seconds.

Suggested tactic:

```coq
Ltac denot_relevant_support_solver :=
  unfold relevant_lvars in *;
  cbn [context_ty_lvars context_ty_lvars_at
       tm_lvars tm_lvars_at value_lvars_at] in *;
  rewrite ?tm_lvars_at_tret_bound0_under in *;
  rewrite ?context_ty_lvars_at_shift_under in * by lia;
  repeat match goal with
  | H : _ ⊆ _ |- _ => specialize_subset_hyp H
  end;
  better_set_solver.
```

But I recommend using it locally first.  Globalizing it too early may make
proof search slower.

### 2. Open-env denotation normalizer

The following sequence appears repeatedly:

```coq
cbn [ty_denote_gas].
rewrite formula_open_env_and.
rewrite formula_open_env_denot_guard by ...
rewrite open_cty_env_* by ...
cbn [ty_denote_gas].
```

Suggested local tactic in `TypeDenote.v`:

```coq
Ltac open_env_denot_prefix :=
  cbn [ty_denote_gas];
  rewrite formula_open_env_and;
  rewrite formula_open_env_denot_guard by eauto;
  cbn [ty_denote_gas].
```

### 3. Guard packet tactic

Several proofs need to turn a guard model into the exact nested `FAnd` model
again.  A tactic is safer than a huge named lemma:

```coq
Ltac denot_guard_as_model Hguard Hname :=
  assert (Hname : _) by (repeat rewrite res_models_and_iff; exact Hguard).
```

Use this in `TypeDenoteOpen.v` and `Soundness.v`.

### 4. Binder packet tactic

The basic typing and instantiate proofs repeatedly do:

- choose fresh `y`;
- assert `HyL`;
- insert typed store;
- rewrite msubst/open under binder.

This should be a small local Ltac in `BasicTypingFormula.v` and possibly
`TermOpen.v`, not a global solver.

## What Not To Do

- Do not delete `ty_denote_gas_tm_result_equiv` or the new compact TLet
  route; they are valuable and currently working.
- Do not replace the long structural induction proofs with one large Ltac.
  That will make future failures opaque.
- Do not extract helpers with huge premises just to reduce line count.  If a
  helper statement is longer than the repeated proof block, prefer a local
  proof packet.
- Do not move denotation-level facts into `ContextBasicDenotation` if they
  mention `ty_denote_gas`; keep dependency direction clean.

## Verification Commands Used

Long proof scan:

```sh
perl -ne '
  if ($ARGV ne $last) { $last=$ARGV; $line=0; }
  $line++;
  if (/^(Lemma|Theorem|Corollary|Proposition|Fact|Instance)\s+([^\s(:]+)/) {
    $pending_name=$2; $pending_line=$line; $pending_file=$ARGV;
  }
  if (/^Proof\./) {
    $in=1; $proof_start=$line; $name=$pending_name;
    $file=$ARGV; $stmt=$pending_line;
  }
  if ($in && /^(Qed\.|Admitted\.)/) {
    $len=$line-$proof_start+1;
    if ($len > 200) {
      print "$len\t$file\t$stmt\t$name\t$proof_start\t$line\n";
    }
    $in=0;
  }
' $(find Denotation ContextTyping ContextBasicDenotation -name '*.v' -maxdepth 3)
```

Near-miss scan used threshold `> 120`.
