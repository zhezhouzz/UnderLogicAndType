# Fix Proof Refactor Analysis

Scope: `ContextTyping/theories/SoundnessFix*.v`.

Goal: inspect every Fix-related lemma for repeated proof patterns, short wrappers
that should be inlined, lower-layer helper candidates, huge intermediate
assertions, and naked set/store proof scripts. This report intentionally keeps
semantic proof architecture intact: the current well-founded Fix proof is
stable and should not be reshaped unless a helper boundary is clearly better.

## Global Findings

- `SoundnessFixSelf.v` contains the most recent proof work and therefore the
  most proof-engineering residue. The semantic structure is now good:
  world-minimum induction, smaller recursive argument extraction, child context
  construction, body application, and unfold transport are separate. The cleanup
  target is mostly side-condition plumbing, not proof architecture.
- Several lemmas are one-use wrappers. Only the very short/wrapper-shaped ones
  should be inlined now; longer one-use lemmas such as open-argument
  normalization are useful semantic checkpoints and should remain named.
- A few helpers are genuinely lower-layer:
  `res_le_store_lift` is a generic resource/refinement fact and belongs in
  `ContextAlgebra`, not in Fix soundness.
  `res_fiber_from_projection_of_store` is also generic but touches the concrete
  fiber constructor; it can be moved later to the resource-extension/fiber layer.
- Naked `better_set_solver` appears frequently in Fix proofs. Most are small
  freshness/domain side conditions. The right cleanup is to hide repeated
  shapes behind local side-condition tactics and small domain lemmas, not to
  let global set solvers run over large semantic contexts.
- There are two plain `set_solver` calls left in Fix files. They should be
  replaced immediately because they are both tiny contradictions.
