Soundness/Fix refactor checklist
================================

Goal
----

Recover the paper-facing Fix/FixD rules and make the semantic case helpers
look like the TLet helper: Denotation-level lemmas should talk about
`denot_ty_lvar_gas` facts and operational/basic premises, not about
`denot_ctx_under` or the typing induction context.

1. Core well-founded relation
-----------------------------

- Add a base-indexed default well-founded order on constants.
- Follow the HAT style: define a measure and use `well_founded_ltof`.
- Keep this at the CoreLang layer because it is about constants/base types,
  not context types.

2. Type-level decreasing qualifier
----------------------------------

- Add a type qualifier expressing that the recursive-call argument is smaller
  than the current argument:

    `mk_q_lt_base b (vbvar 0) (vfvar x)`

- Add a small recursive-call type abbreviation:

    `fix_rec_call_ty b x τx τ :=
       CTArrow (CTInter τx (over_ty b (mk_q_lt_base b (vbvar 0) (vfvar x)))) τ`

  The current CoreLang `vfix` encoding opens the ordinary argument first; the
  opened value is still a function waiting for the recursive self reference.

3. Typing rules
---------------

- Replace the current unrestricted Fix/FixD rules.
- Add premises:

    `erase_ty τx = TBase b`
    `erase_ty τ = t`

- In the body premise, recursive self gets the decreasing call type.

4. Denotation case helpers
--------------------------

- Remove the proof-facing `*_direct_denotation_gas_in_ctx` interface.
- Introduce direct gas-level helpers:

    `letd_intro_denotation`
    `lam_intro_denotation`
    `lamd_intro_denotation`
    `app_elim_denotation`
    `appd_elim_denotation`
    `fix_intro_denotation`
    `fixd_intro_denotation`
    `appop_intro_denotation`
    `match_both_intro_denotation`
    `match_true_intro_denotation`
    `match_false_intro_denotation`

- Already-proved cases may keep their proofs after renaming.
- Unproved cases remain named obligations, not anonymous inline proof holes.

5. Soundness layer
------------------

- Soundness wrappers consume `denot_ctx_under` and IHs.
- They call the new Denotation-level helpers.
- Fix/FixD wrappers pass the new erasure premises and decreasing recursive
  self type.

6. Verification
---------------

- Compile the affected files first.
- Then compile enough of the repo to ensure the rule/interface migration is
  mechanically coherent.
